# Проект: Исследовательский анализ данных объявлений о продаже квартир

## 1. Назначение и цель проекта

Разработать методологию определения рыночной стоимости квартир на основе исторических данных сервиса «Яндекс Недвижимость».  
Результаты анализа должны стать основой для автоматизированной системы выявления аномалий и мошеннических объявлений.

## 2. Источник данных

Архив объявлений о продаже квартир в Санкт-Петербурге и соседних населённых пунктах за несколько лет.  
Файл: `real_estate_data.csv`

## 3. Типы данных в записи

По каждой квартире доступны два вида данных:
- **Введённые пользователем** (общая площадь, количество комнат, этаж и т.д.)
- **Автоматические** (на основе геоданных):  
  расстояние до центра, аэропорта, до ближайшего парка, количество парков и водоёмов в радиусе.

## 4. Задачи исследования

### 4.1. Предобработка данных
- Выявить и обработать пропуски (с обоснованием замены или сохранения).
- Привести типы данных в столбцах к корректным.
- Устранить неявные дубликаты в названиях населённых пунктов.

### 4.2. Расчёт новых признаков
Создать столбцы:
- Цена за 1 м²
- День недели публикации
- Месяц публикации
- Год публикации
- Тип этажа: первый, последний, другой
- Расстояние до центра города (км, округлённое до целых)

### 4.3. Исследовательский анализ

**4.3.1. Распределение параметров**
Построить гистограммы для:
- общей, жилой площади и площади кухни
- цены объекта
- количества комнат
- высоты потолков
- типа этажа
- количества этажей в доме
- расстояния до центра
- расстояния до ближайшего парка

*Примечание:* редкие и выбивающиеся значения должны быть обработаны или исключены перед визуализацией.

**4.3.2. Анализ срока продажи (`days_exposition`)**
- Построить гистограмму
- Рассчитать среднее и медиану
- Определить типичный, быстрый и аномально долгий срок экспозиции

**4.3.3. Факторы влияния на цену**
Оценить зависимость полной стоимости квартиры от:
- общей, жилой площади и площади кухни
- количества комнат
- типа этажа (первый / последний / другой)
- дня недели, месяца и года публикации

**4.3.4. Стоимость квадратного метра по локациям**
- Выделить 10 населённых пунктов с наибольшим числом объявлений
- Построить сводную таблицу: количество объявлений и средняя цена за м²
- Определить локации с самой высокой и низкой стоимостью м²

**4.3.5. Анализ для Санкт-Петербурга**
- Отфильтровать объявления по Санкт-Петербургу
- Рассчитать среднюю стоимость квартир для каждого километра удаления от центра
- Построить график зависимости цены от расстояния до центра

## 5. Входные данные (структура)

| Поле | Описание |
|------|----------|
| `airports_nearest` | расстояние до ближайшего аэропорта (м) |
| `balcony` | число балконов |
| `ceiling_height` | высота потолков (м) |
| `cityCenters_nearest` | расстояние до центра города (м) |
| `days_exposition` | длительность размещения объявления (дней) |
| `first_day_exposition` | дата публикации |
| `floor` | этаж |
| `floors_total` | всего этажей в доме |
| `is_apartment` | апартаменты (да / нет) |
| `kitchen_area` | площадь кухни (м²) |
| `last_price` | цена на момент снятия с публикации |
| `living_area` | жилая площадь (м²) |
| `locality_name` | населённый пункт |
| `open_plan` | свободная планировка |
| `parks_around3000` | число парков в радиусе 3 км |
| `parks_nearest` | расстояние до ближайшего парка (м) |
| `ponds_around3000` | число водоёмов в радиусе 3 км |
| `ponds_nearest` | расстояние до ближайшего водоёма (м) |
| `rooms` | число комнат |
| `studio` | студия |
| `total_area` | общая площадь (м²) |
| `total_images` | количество фотографий в объявлении |

## 6. Выходные результаты

- Очищенный и структурированный набор данных с новыми расчётными признаками
- Набор графиков и сводных таблиц по каждому из пунктов раздела 4
- Выводы о факторах, значимо влияющих на рыночную стоимость
- Рекомендации для построения автоматической системы обнаружения аномалий
